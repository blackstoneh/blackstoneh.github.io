在Github创建blog流程

首先我的系统是win10或win7，非典型程序猿嘛，抱着win的大腿混口饭吃得了，懒得研究啥Linux、Unix。。想着都头大，最好以后我用不到才好，不过谁知道呢，用到的时候再说吧，嘿嘿~
下面是我的流程

  • 注册Github帐号

  • 在Github上，创建”用户名.github.io”名称的repository（注意，这里的用户名一定要和github上注册的用户名相同）

  • 安装Node.js，安装git，安装github桌面版

  • 上面三个软件下载安装完成后，运行”git bash”来安装hexo，命令是

    npm install -g hexo-cli

  • 在本地新建作为【站点的文件夹】

  • 运行【命令提示符】窗口，并cd至【站点文件夹】，然后执行命令进行初始化（后面称【窗口1】）

    hexo init
    npm install

  • 完成后就可以启动服务了

    hexo server

  • 在浏览器中输入http://localhost:4000/，如果出现hello页面说明服务器已正确运行

  • 再新打开一个【命令提示符】窗口，并cd至【站点文件夹】，然后执行命令进行初始化（后面称【窗口2】）

  • 为了在发布文章中方便插入本地图片，首先检查【站点文件夹】下的_config.yml 中有 post_asset_folder:true，没有加上，这样就可以在新建文章的时候同时生成资源文件夹；【窗口2】下执行命令

    npm install https://github.com/CodeFalling/hexo-asset-image --save
    这样在文章中就可以使用 ![图片名称](资源文件夹/图片名称.jpg)这类方式引入本地图片了

  • 发布新文章，【窗口2】执行命令

    hexo new "first blog"
    命令执行后会在【站点文件夹】\source_posts 中出现first blog.md的文件和文件夹

  • 打卡first blog.md文件进行编辑，可以看到上面---内的内容是文章的一些信息标记，可以进行编辑，文章的内容写在---之后，使用的是markdown格式，可以网上编辑器进行编辑，完成后保存

  • 【窗口2】执行命令重新生成静态页面

    hexo generate

  • 【窗口1】执行停止服务后重新执行’hexo server’开启服务，然后访问http://localhost:4000/，就可以看到新发布的文章了

  • 为了发布到github上，编辑【站点文件夹】下的_config.yml，修改
    deploy: type: git repository: https://github.com/你的github名称/你的github名称.github.io.git branch: master
    修改保存后在【窗口2】执行

    hexo generate
    hexo deploy
    在执行完第二个名录时，如果安装了github桌面版，会弹出登陆框，输入你的github帐户密码，成功后就可以正常部署了

  • 浏览器中输入 https://你的github名称.github.io，就可以欣赏你的blog了^_^

当然，上面只是我部署blog的大致流程，由于是部署完成后根据印象写的，所以有可能有些地方会有些问题，如果有人发现或是有不清楚的，留言吧。。。
另外，博客里还可以增加评论、搜索、rss、打赏等插件，可以用到的时候再搜索相关文章，网上也多的是，我就不介绍了~
其实这篇文章做的比较粗糙，本来也不是作为教学用的，主要是怕自己忘了，给自己留个印象！不过现在技术发展的这么快，估计也不一定用的上了，哈哈
